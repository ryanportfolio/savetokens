Token-saving techniques for AI coding agents

savetokens.tips · datasheet ST-0001

Token-saving techniques for AI coding agents are methods that reduce shell-output context while preserving decision-relevant information. These token-saving techniques apply to Claude Code, Codex, and other terminal-based coding agents. Measured figures come from logged raw-versus-filtered token deltas; estimated figures carry no counterfactual.

Application Information

5.1 RTK-DIFF-01, route git diff through the RTK proxy

Functional block diagram of one proxied command. Raw diff enters, RTK filters it, a compact diff summary leaves. The delta is the characterized parameter.

M, rtk gain, 2026-07-19, global, n=3

A full git diff of a large change can send millions of characters to the model, most of which the agent never reads line by line. Routing the command through RTK returns a filtered summary that preserves the decision content: which files changed, the shape of the change, and the parts that need review.

In one measured large diff the filtered output used 99.2 percent fewer tokens than the raw diff, a saving of 5.6 million tokens. The delta is pooled across three runs and treated as one sample, not a stable average, so read it as a strong signal rather than a guaranteed rate. Use rtk git diff whenever a diff is large enough that you would scroll past most of it anyway.

!

Frequently Asked Questions

How much does routing commands through RTK reduce token usage?

Measured across 93,501 proxied commands in the snapshot dated 2026-07-19, the RTK proxy techniques documented on savetokens.tips reduced output tokens by 82.5 percent overall, 19.1 million tokens saved. Per-command savings range from 22.5 percent for rtk grep to 99.2 percent for one large rtk git diff. Every figure is characterized in Section 3 with sample counts and test conditions.

What is the difference between a measured and an estimated figure?

A measured figure, tagged [M], has a real counterfactual: raw output tokens versus RTK-filtered output tokens, logged per command. An estimated figure, tagged [E], is a design target with no counterfactual, prints coarse with a leading tilde, and never carries the accent color. The two kinds are held apart on six channels at once, readable in grayscale.

Which commands save the most tokens?

Commands with large raw output filter best: rtk git diff saved 99.2 percent (n=3), rtk vitest run saved 92.0 percent (n=42), rtk ls -la saved 53.8 percent (n=19). High-frequency commands save a lower percentage but large absolute totals: rtk read saved 24.5 percent, 9.5 million tokens, across 626 runs.

Does savetokens.tips apply to both Claude Code and Codex?

Yes. Every technique listed in Ordering Information applies to both Claude Code and Codex. The RTK proxy sits between the agent and the shell, so the token filtering is agent-agnostic.
